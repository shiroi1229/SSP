# Chat UI 現状評価（2025-xx）

## 確認対象
- リポジトリ内のフロントエンド実装状態
- 既存の PoC レポート (`ui_v0_5_poc_report.md`) と PoC ログ (`logs/ui_v0_5_poc.jsonl`) に基づく UI 連携状況

## 観察結果
1. **フロントエンドの実装がリポジトリに存在しない**
   - `frontend/` 配下にソースファイルがなく、現状のリポジトリだけではチャット画面 UI を再現できない。
2. **PoC では UI 操作は可能だがバックエンド連携が未整備**
   - UI-v0.5 のレポートでは評価パネルと RAG Source Graph が画面上で操作・描画可能とされている一方、`/api/chat` で 500、RAG 検索で 404 を返しており、API 連携が成立していない。
3. **PoC ログでも API エラーが再現**
   - 最新の PoC ログでもチャット API が 500、検索 API が 404 となっており、バックエンド側の修正が未着手であることを示している。

## 評価
- 現行リポジトリのみではチャット UI を実際に評価するためのフロントエンド資材が不足しており、UI 体験の確認・改善が行えない。
- PoC ドキュメント上では UI コンポーネント（EvaluationPanel、RAG Source Graph）は動作するとされるが、API エラーにより実データと連動した評価フローが成立していない。

## 推奨アクション
1. **フロントエンド資材の復元または取得**: `frontend/` に React コンポーネント等を復元し、ローカルで画面表示を再現できる状態にする。
2. **API 連携の修正**: `/api/chat` の 500 と `/api/knowledge/search` の 404 を解消し、UI からの送信〜検索〜評価の一連のフローを結線する。
3. **評価ツールの再実行とスクリーンショット取得**: バックエンド修正後に PoC ツールを再実行し、UI のスクリーンショットとともに実測値を更新する。
